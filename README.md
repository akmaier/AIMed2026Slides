# AIMed 2026 Slides

Slides for the talk **"What next in medical AI?"** by Andreas Maier
(FAU Erlangen-N&#252;rnberg) at the AIMed 2026 conference,
Krak&#243;w, 7-9 May 2026.

The compiled deck is `output_tex/main.pdf` (49 pages). Source is
`output_tex/main.tex`, built against the FAU beamer template (the
`styles/` and `template-art/` directories are vendored under
`output_tex/`).

## Build

```
cd output_tex
latexmk -pdf main.tex
```

A single `latexmk` run handles the biber bibliography pass automatically.
If you prefer to drive the engines by hand:

```
pdflatex main.tex
biber    main
pdflatex main.tex
pdflatex main.tex
```

The deck is self-contained: no `animate` package, no extracted frame
sequences, and the QR code on the Thank-you slide is a pre-rendered PNG
(`output_tex/figures/qr_github.png`) - so any PDF reader displays the
deck identically.

## Regenerating the QR code

The QR code is generated by a small Python script using the `qrcode`
package (pure black on white, error correction M):

```
python3 generate_qr.py
```

Edit `URL` at the top of `generate_qr.py` if the target ever changes,
re-run the script, then rebuild the deck.

## Contents

* `output_tex/main.tex` -- LaTeX source
* `output_tex/main.pdf` -- compiled deck
* `output_tex/bibliography.bib` -- biblatex entries (loaded by the FAU theme)
* `output_tex/figures/` -- images, including `qr_github.png` and
  the static `img_08_0*.png` renders that replaced the original GIFs
* `output_tex/styles/`, `output_tex/template-art/` -- FAU beamer template
* `generate_qr.py` -- QR-code generation script
* `slide_check.md` -- typesetting / translation report

The original PowerPoint, the rasterised slide previews and the
`extracted_pptx/` JSON dump are kept locally for traceability but are
excluded from this repository via `.gitignore`.
